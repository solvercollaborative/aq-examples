AQ Authenticated Embed

This example shows how to authenticate users with the AQ chatbot. Your backend generates a signed JWT token, which the frontend passes to the widget.

Simulate Login

In production, this would be your real login form. The important part is calling your backend to get a JWT token after authentication.

Authenticated as . The chatbot now has access to member-specific content.

How It Works

  1. User logs in via your website's auth system
  2. Your backend calls POST /api/aq-token to generate a JWT
  3. Frontend receives the token and sets it on the widget: 
    window.AQWidget.setAuthToken(token);
  4. The widget sends the token with every request via X-AQ-Auth-Token header
  5. AQ validates the signature and grants access to member-scoped content
